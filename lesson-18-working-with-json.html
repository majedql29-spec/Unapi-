Lesson 18: Working with JSON

Intermediate⏰ 8 min

JSON (JavaScript Object Notation) is the universal data interchange format for web APIs, configuration files, and data storage. Python's built-in json module makes reading, writing, and transforming JSON effortless. Understanding JSON is essential for almost any modern Python application that communicates over the internet.

import json

person = {
    "name": "Alice",
    "age": 30,
    "skills": ["Python", "Data Analysis"]
}

json_str = json.dumps(person, indent=2)
print(json_str)

parsed = json.loads(json_str)
print("Name:", parsed["name"])

Core functions — strings vs files:

  • json.dumps() — converts a Python object to a JSON string. Use indent for pretty-printing: json.dumps(data, indent=2, sort_keys=True). Control encoding with ensure_ascii=False to preserve non-ASCII characters like emojis. The default parameter specifies a function for objects that aren't natively serializable (like datetime or Decimal).
  • json.loads() — parses a JSON string back into a Python object. Returns a dict for JSON objects, list for JSON arrays, etc.
  • json.dump(obj, file) — writes directly to a file object. No s suffix: with open("data.json", "w") as f: json.dump(data, f, indent=2).
  • json.load(file) — reads directly from a file object: with open("data.json") as f: data = json.load(f).

JSON to Python type mapping:

  • JSON object {} → Python dict
  • JSON array [] → Python list
  • JSON string → Python str
  • JSON number → Python int or float
  • JSON boolean true/false → Python True/False (note capitalization)
  • JSON null → Python None

Handling nested JSON: Real-world JSON often has multiple levels of nesting. Access nested data by chaining keys and indices: data["users"][0]["address"]["city"]. For safety, use .get() at each level: data.get("users", [{}])[0].get("address", {}).get("city"). For extremely deep nesting, consider using jsonpath-ng library or flatten the structure with recursive helper functions.

Error handling with JSON: Always wrap JSON parsing in try/except because the input might be malformed: try: data = json.loads(raw_text); except json.JSONDecodeError as e: print(f"Invalid JSON: {e}"). json.JSONDecodeError provides .msg, .doc, .pos, .lineno, and .colno for detailed error reporting. Also handle TypeError when serializing objects that aren't JSON-serializable (like datetime objects — use the default parameter with a custom converter).

Common mistakes: Forgetting that JSON uses true/false/null (lowercase) while Python uses True/False/None — the json module handles this conversion automatically. Trying to serialize non-serializable types (datetimes, Decimals, custom objects) — provide a default converter. Using single quotes in JSON strings — JSON requires double quotes. Trailing commas in JSON objects or arrays — JSON does not allow them.

Best practices: Always specify indent=2 for human-readable output. Use ensure_ascii=False to preserve Unicode characters. Validate JSON before processing by catching JSONDecodeError. For large JSON files, consider streaming parsers like ijson. Use sort_keys=True for consistent, diff-friendly output in version-controlled config files.

Real-world use case: Fetching data from a public API: import urllib.request; response = urllib.request.urlopen("https://api.github.com/users/python"); data = json.loads(response.read()); print(data["login"], data["public_repos"]). Or storing application configuration: read a config.json with json.load() and access settings like database credentials or feature flags.

Try it: Add ensure_ascii=False to the dumps call and include a Unicode string like "café". Then try serializing a nested structure and use .get() to safely access a deeply nested key. Finally, wrap the loads call in try/except and pass intentionally invalid JSON to see the error message.
