Lesson 22: CSV File Handling

Intermediate⏰ 8 min

CSV (Comma-Separated Values) is a universal file format for tabular data — exported by Excel, Google Sheets, databases, and countless other tools. Python's built-in csv module provides robust support for reading and writing CSV files, handling edge cases like embedded commas, quoted fields, and different line-ending conventions.

Two approaches: reader/writer vs DictReader/DictWriter

  • csv.reader / csv.writer — work with rows as lists. Simple and fast, but you must remember column positions: row[0], row[1], etc. This is fragile — if columns are reordered, your code breaks silently.
  • csv.DictReader / csv.DictWriter — work with rows as dictionaries keyed by header names. Much more readable and robust: row["Name"], row["Age"]. If columns are reordered, your code still works. DictWriter also automatically writes the header row via .writeheader(). DictReader automatically uses the first row as headers. This is the recommended approach for most production code.
import csv

# Writing a CSV
with open("data.csv", "w", newline="") as f:
    writer = csv.writer(f)
    writer.writerow(["Name", "Age", "City"])
    writer.writerow(["Alice", 30, "New York"])
    writer.writerow(["Bob", 25, "London"])

# Reading a CSV
with open("data.csv", "r") as f:
    reader = csv.reader(f)
    for row in reader:
        print(row)

# Using DictReader/DictWriter (recommended)
with open("data.csv", "r") as f:
    reader = csv.DictReader(f)
    for row in reader:
        print(row["Name"], row["Age"])

# Writing with DictWriter
with open("output.csv", "w", newline="") as f:
    fieldnames = ["Name", "Age", "City"]
    writer = csv.DictWriter(f, fieldnames=fieldnames)
    writer.writeheader()
    writer.writerow({"Name": "Charlie", "Age": 35, "City": "Paris"})

Handling headers

DictReader reads the first row as fieldnames automatically. If your CSV has no header, pass fieldnames=["col1", "col2"] to the constructor. DictWriter requires you to specify fieldnames — this determines both the column order in the output and which keys are written. Call .writeheader() to write the header row.

Custom delimiters

Not all CSVs use commas! Tab-separated files (TSV) use delimiter="\t". European formats sometimes use semicolons: delimiter=";". Always inspect the file first and specify the delimiter explicitly: csv.reader(f, delimiter=";").

Quoting control

The quoting parameter controls when fields are quoted:

  • csv.QUOTE_MINIMAL (default) — quote only when necessary (fields containing delimiter, quotechar, or newlines)
  • csv.QUOTE_ALL — quote every field. Compatible with more parsers but produces larger files.
  • csv.QUOTE_NONNUMERIC — quote all non-numeric fields. Reader converts unquoted fields to floats.
  • csv.QUOTE_NONE — never quote. May produce invalid CSV if fields contain special characters.

Large file processing

For files with hundreds of thousands of rows, never read the entire file into memory. Process row by row using a for loop:

with open("large.csv", "r", encoding="utf-8") as f:
    reader = csv.DictReader(f)
    for row in reader:
        process_row(row)  # One row at a time

This uses constant memory regardless of file size. For very large CSV processing, consider pandas.read_csv(chunksize=10000).

Common mistakes

Forgetting newline="" when opening files for writing — on Windows, this adds extra blank lines between rows. Not specifying encoding="utf-8" — causes UnicodeDecodeError with international characters. Assuming every row has the same number of columns — always validate or handle ValueError. Not accounting for header rows — DictReader skips them automatically, reader does not.

Best practices

Always use newline="" in file open when working with CSV. Always specify encoding="utf-8". Prefer DictReader/DictWriter over reader/writer for clarity. Handle potential ValueError for malformed rows. For cross-platform CSV, use lineterminator="\r\n" in writer. Check for empty cells: if row.get("column"): handles missing or empty values gracefully.

Real-world use case

Processing a sales report: with open("sales.csv", "r", encoding="utf-8") as f: reader = csv.DictReader(f); total = sum(float(row["Amount"]) for row in reader if row["Status"] == "Completed"). Or generating a report: export user data from a database to CSV for download, using DictWriter to write headers and rows with proper quoting and UTF-8 encoding.

Try it: Add a row to the CSV using DictWriter instead of writer. Then read it back with DictReader and access columns by name. Try changing the delimiter to ";" and see how the output changes. Finally, add quoting=csv.QUOTE_ALL and observe the difference in the output file.
