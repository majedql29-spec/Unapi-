Lesson 10: Sets & Tuples

Beginner⏰ 9 min

Beyond lists, Python offers two more essential collection types: tuples and sets. Each has specific use cases that make them invaluable in different situations. Choosing the right collection type is a key skill in writing idiomatic Python.

Tuples are ordered sequences just like lists, but with one critical difference — they are immutable. Once created, a tuple cannot be changed: you cannot add, remove, or replace elements. This immutability makes tuples useful for data that should remain constant throughout your program, such as geographic coordinates (latitude, longitude), RGB color values (255, 128, 0), or configuration settings that must not be accidentally modified. Tuples are created with parentheses: point = (3, 4). You access elements by index just like lists: point[0] gives 3. You can also unpack a tuple into individual variables: x, y = point assigns 3 to x and 4 to y. This unpacking feature is used extensively in Python for clean, readable code.

Tuples are also hashable (if all their elements are hashable), which means they can be used as dictionary keys — something lists cannot do. For example, you can use a tuple as a key in a dictionary to store values by coordinates: locations[(lat, lon)] = "Home".

Sets are unordered collections of unique elements. The most important feature of a set is that duplicates are automatically removed. Sets are created with curly braces: fruits = {"apple", "banana", "cherry", "apple"} — the second "apple" is silently ignored. Because sets are unordered, you cannot access elements by index. However, this lack of ordering gives sets super-fast membership testing: checking if an element is in a set ("apple" in fruits) is much faster than checking a list, especially for large collections.

Set operations — mathematically powerful:

  • Union (|) — combines elements from both sets, removing duplicates: {1, 2} | {2, 3} gives {1, 2, 3}
  • Intersection (&) — elements present in both sets: {1, 2, 3} & {2, 3, 4} gives {2, 3}
  • Difference (-) — elements in the first set but not in the second: {1, 2, 3} - {2, 3} gives {1}
  • Symmetric difference (^) — elements in either set but not both: {1, 2} ^ {2, 3} gives {1, 3}

Practical analogy: Think of a tuple as a sealed envelope (you can see what's inside but cannot change it), a set as a bag of unique marbles (no duplicates, fast to search), and a list as a shelf of books (ordered, you can rearrange and add more).

Common mistake: Creating an empty set with {} actually creates an empty dictionary. To create an empty set, use set(). And remember that tuples with a single element need a trailing comma: (1,), not just (1) which is just a number in parentheses.

# Tuple (immutable)
point = (3, 4)
print("x:", point[0], "y:", point[1])

# Set (unique, unordered)
fruits = {"apple", "banana", "cherry", "apple"}
print("Set:", fruits)

# Set operations
a = {1, 2, 3, 4}
b = {3, 4, 5, 6}
print("Union:", a | b)
print("Intersection:", a & b)
print("Difference:", a - b)
Try it: Add "apple" in fruits to check membership, and print the result. Then try the symmetric difference operator ^ between sets a and b. Finally, try using a tuple as a dictionary key: coords = {(0, 0): "origin", (1, 0): "right"} and access coords[(0, 0)].
