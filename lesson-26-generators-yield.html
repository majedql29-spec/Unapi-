Lesson 26: Generators & Yield

Advanced⏰ 9 min

Generators produce sequences of values lazily — they yield one value at a time instead of computing and storing all values at once. This makes them exceptionally memory-efficient, especially for large or infinite datasets. A generator function uses yield instead of return. When called, it returns a generator object without executing the function body. Each time you request the next value via next() or a for loop, the function runs until it hits yield, produces the value, and pauses — preserving its entire state for the next call.

def count_up_to(n):
    i = 1
    while i <= n:
        yield i
        i += 1

for num in count_up_to(5):
    print(num)

# Generator expression
squares = (x**2 for x in range(5))
print(list(squares))

Generator expressions vs list comprehensions

Both have the same syntax except for brackets. A list comprehension [x**2 for x in range(10_000_000)] would allocate a 10-million-element list, consuming hundreds of megabytes of memory. A generator expression (x**2 for x in range(10_000_000)) uses almost no memory — it computes each value on demand. Use generator expressions when you only need to iterate once and the dataset is large. Use list comprehensions when you need random access, need to iterate multiple times, or the dataset is small enough to fit in memory comfortably.

yield from — delegating to sub-generators

Python 3.3+ introduced yield from for yielding all values from another iterable or generator. It simplifies chaining or flattening generators:

def chain(*iterables):
    for it in iterables:
        yield from it  # equivalent to: for item in it: yield item

send(), throw(), and close() — bidirectional generators

Generators can receive values via .send(value), which becomes the return value of yield inside the generator:

def echo():
    while True:
        received = yield
        print(f"Got: {received}")

g = echo()
next(g)  # Prime the generator (advance to first yield)
g.send("Hello")  # Prints "Got: Hello"

.throw(exception_type) raises an exception at the yield point inside the generator. .close() sends a GeneratorExit exception, causing the generator to exit.

Practical use cases for generators

Processing large files line-by-line without loading the entire file into memory. Generating infinite sequences like Fibonacci numbers. Implementing data processing pipelines where each stage is a generator that reads from the previous stage. Streaming API responses. Implementing custom iterators for complex traversal patterns (tree traversal, graph search).

Common mistakes

Forgetting to prime a generator with next() before .send() — raises TypeError. Using return value in a generator — in Python 3, return value provides the value in a StopIteration exception. Creating a generator but never consuming it — it does nothing until iterated. Using a list comprehension when a generator expression would be more memory-efficient for large datasets.

Best practices

Use generator expressions for one-time iteration over large datasets. Use generator functions for complex sequences that require state or logic. Use yield from for delegating to sub-generators. Remember that generators are single-use — once exhausted, they cannot be restarted. For expensive-to-create generators, consider using itertools.tee() if you need multiple independent iterators.

Real-world use case

A log processing pipeline: def read_logs(path): yield from open(path); def filter_errors(lines): for line in lines: if "ERROR" in line: yield line; def extract_fields(lines): for line in lines: yield parse_log_line(line). Each stage is a generator that lazily processes data from the previous stage, allowing processing of gigabyte-sized log files with constant memory usage.

Try it: Write a generator that yields the first N Fibonacci numbers. Then use yield from to concatenate two sequences. Finally, use .send() to create a generator that accumulates values: def accumulator(): total = 0; while True: total += yield total and send it numbers to see running totals.
