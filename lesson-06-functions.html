Lesson 06: Functions

Beginner⏰ 10 min

Functions are reusable blocks of code that perform a specific task. They are the primary mechanism for organizing code, avoiding repetition (following the DRY principle — Don't Repeat Yourself), and making programs easier to read, test, and maintain. Think of a function as a mini-program within your program: it takes inputs, does work, and optionally produces an output.

Define a function with the def keyword, followed by the function name, parentheses that may contain parameters, and a colon. The function body is indented by one level (typically 4 spaces):

def greet(name):
    """Return a personalized greeting."""
    return f"Hello, {name}!"

Functions can accept parameters (also called arguments) that serve as inputs. They can return a value using the return keyword. The return statement immediately exits the function and sends the specified value back to the caller. If a function does not have a return statement, it implicitly returns None (Python's null value).

Once defined, you call a function by its name followed by parentheses containing any required arguments: message = greet("JometCode"). The returned value can be stored in a variable, printed, or used in an expression.

Parameter features to know:

  • Default parameters: You can assign default values: def greet(name, greeting="Hello"): — if the caller omits greeting, it defaults to "Hello"
  • Keyword arguments: Call functions by naming arguments: greet(name="JometCode", greeting="Hi") — order doesn't matter with keyword arguments
  • Variable-length arguments: Use *args for any number of positional arguments and **kwargs for any number of keyword arguments

Best practices for writing good functions:

  • Give functions descriptive names that explain what they do — calculate_average() is better than calc()
  • Each function should do exactly one thing and do it well (Single Responsibility Principle)
  • Keep functions short — if a function exceeds 20-30 lines, consider breaking it into smaller functions
  • Add a docstring (triple-quoted string) right after the definition to document what the function does

Common mistake: Forgetting to call the function (missing parentheses). Writing greet without parentheses returns the function object itself, not the result. Always use greet("name") with parentheses to actually execute the function.

def greet(name):
    return f"Hello, {name}!"

def add(a, b):
    return a + b

print(greet("JometCode"))
print(add(5, 3))
Try it: Add a third function that takes three parameters and returns the average. Then call it and print the result. Experiment with default parameter values.
