Lesson 13: Dictionaries

Intermediate⏰ 8 min

Dictionaries store data as key-value pairs, making them one of Python's most versatile data structures. Each key is unique and maps to a value. Think of a real dictionary — you look up a word (key) and instantly get its definition (value). In code: user = {"name": "Alice", "age": 30}. Keys must be immutable types (strings, numbers, tuples), while values can be any type — lists, other dictionaries, functions, or even None.

Essential dict methods:

  • .get(key, default) — safely access a key without raising KeyError. Returns default (or None) if the key is missing: user.get("email", "no email"). This is the idiomatic way to handle optional keys.
  • .keys() — returns a view of all keys. Useful for iteration or membership checks: "name" in user.keys().
  • .values() — returns a view of all values. Great for aggregation: sum(scores.values()).
  • .items() — returns key-value pairs for unpacking in loops: for k, v in user.items():.
  • .setdefault(key, default) — returns the value if the key exists, otherwise inserts the default and returns it. Perfect for building nested structures.
  • .update(other_dict) — merges another dictionary into this one, overwriting existing keys.
  • .pop(key, default) — removes a key and returns its value (or default if missing).

Nested dictionaries are dictionaries inside dictionaries — a common pattern for representing structured data like JSON API responses:

users = {
  "alice": {"age": 30, "city": "NYC"},
  "bob": {"age": 25, "city": "London"}
}
print(users["alice"]["city"]) # NYC

Access nested values safely by chaining .get(): users.get("alice", {}).get("city"). For deeply nested data, consider using collections.defaultdict or the dict approach with setdefault.

Dict comprehensions offer a concise way to build dictionaries: squares = {x: x**2 for x in range(5)} produces {0: 0, 1: 1, 2: 4, 3: 9, 4: 16}. Add conditions: {x: x**2 for x in range(10) if x % 2 == 0}. You can also transform keys and values: {k.lower(): v for k, v in data.items()}.

When to use dict vs list: Use dictionaries when you need to look up values by a meaningful key (like a name or ID). Use lists when the order of items matters or when you need to store a simple sequence. Dictionaries excel at membership testing (key in dict is O(1) on average) and representing structured records. Lists excel at ordered iteration and sequential access.

Common mistakes: Using dict[key] without checking if the key exists (raises KeyError). Always prefer .get() for optional keys. Another mistake is modifying a dictionary while iterating over it — create a copy with .copy() or iterate over list(dict.keys()) instead. Also beware that {} creates an empty dict, not an empty set (use set() for that).

Best practices: Use descriptive key names (strings or meaningful constants). Prefer .get() over direct indexing for optional keys. Use .items() for iteration to avoid looking up values by key. For counting or grouping, use collections.Counter or defaultdict instead of manual dict logic. Since Python 3.7, dictionaries preserve insertion order — rely on this for predictable iteration.

Real-world use case: Dictionaries are the backbone of JSON data processing. When you call a REST API, the response is almost always parsed into nested dictionaries. Configuration files, user profiles, cached data, and database records are all naturally modeled as dictionaries. They are also used internally by Python for namespaces, class attributes, and instance data.

user = {
    "name": "JometCode",
    "age": 2,
    "language": "Python"
}
print(user["name"])
user["version"] = "1.0"
for key, val in user.items():
    print(f"{key}: {val}")
Try it: Add a nested dictionary with user["address"] = {"city": "Paris", "zip": "75001"} then print user["address"]["city"]. Then try a dict comprehension: {k: v.upper() if isinstance(v, str) else v for k, v in user.items()}. Finally, use .get() with a default to safely access a missing key.
