Lesson 16: File Handling

Intermediate⏰ 8 min

Working with files is a fundamental task in programming. Python's open() function, combined with the with statement, makes file handling safe, simple, and reliable. The with statement creates a context manager that automatically closes the file when the block ends — even if an exception occurs. This is much safer than manually calling .close(), which can be skipped if an error happens before it.

with open("example.txt", "w") as f:
    f.write("Hello from JometCode!")
with open("example.txt", "r") as f:
    content = f.read()
print(content)

File modes determine how you interact with a file:

  • "r" — read (default). The file must exist. Raises FileNotFoundError if missing.
  • "w" — write. Creates a new file or overwrites an existing one (truncates to zero length).
  • "a" — append. Adds content to the end of the file. Creates the file if it doesn't exist. Perfect for logs.
  • "x" — exclusive creation. Creates a new file but raises FileExistsError if it already exists. Safe for preventing accidental overwrites.
  • "r+" — read and write. Opens without truncating. The file must exist.

Adding b to any mode (e.g., "rb", "wb") opens the file in binary mode for non-text files like images, audio files, or compiled data. In binary mode, data is read and written as bytes objects rather than strings. Adding t (e.g., "rt") explicitly opens in text mode (the default).

Reading from files: Beyond .read() which returns the entire content as one string, you can:

  • .readline() — reads one line at a time (including the trailing newline). Memory-efficient for large files.
  • .readlines() — returns a list of all lines. Convenient but loads everything into memory.
  • Iterate directly: for line in file: — the most memory-efficient approach for large files, reading one line at a time without loading the entire file.

Writing: .write(text) writes a string. .writelines(lines) writes a list of strings without adding newlines — you must include them yourself. For appending, use mode "a": with open("log.txt", "a") as f: f.write("New entry\n").

File paths: Use raw strings or double backslashes on Windows to avoid escape sequences: r"C:\Users\name\file.txt" or "C:/Users/name/file.txt" (forward slashes work on Windows too). The os.path module provides path manipulation utilities: os.path.join("folder", "subfolder", "file.txt") builds the correct path for your OS. Modern Python prefers pathlib: from pathlib import Path; path = Path("folder") / "subfolder" / "file.txt" — this is cleaner, cross-platform, and provides convenient methods like .read_text(), .write_text(), .exists(), .suffix, and .stem.

Encoding: Files store bytes; encoding determines how bytes map to characters. Always specify the encoding when reading/writing text files: open("file.txt", "r", encoding="utf-8"). UTF-8 is the modern standard and supports every Unicode character. Omitting encoding can lead to UnicodeDecodeError on systems with different default encodings (like Windows, which may use cp1252).

Common mistakes: Forgetting to close a file (solved by with). Not specifying encoding — leads to cross-platform bugs. Using .read() on a file that's too large to fit in memory — use line-by-line iteration instead. Forgetting that "w" overwrites existing content without warning — use "x" mode or check os.path.exists() first. Not handling FileNotFoundError when opening files for reading.

Best practices: Always use with open(...) as f: — never call .close() manually. Always specify encoding="utf-8" for text files. Use pathlib.Path for modern, clean path handling. Process large files line-by-line rather than reading the entire file at once. Wrap file operations in try/except to handle FileNotFoundError, PermissionError, and IOError.

Real-world use case: Processing a 2GB server log file: with open("server.log", "r", encoding="utf-8") as f: for line in f: if "ERROR" in line: error_count += 1 — this processes the file line-by-line, using minimal memory regardless of file size. For writing: an application that logs user activity to activity.log in append mode, creating a new log file each day with the date as the filename using pathlib.

Try it: Change the code to read the file line by line using a for loop. Then try append mode "a" to add a second line and read the file again. Finally, use pathlib.Path to read the file: from pathlib import Path; print(Path("example.txt").read_text()).
