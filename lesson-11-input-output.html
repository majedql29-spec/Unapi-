Lesson 11: Input & Output

Beginner⏰ 6 min

The input() function is Python's built-in way to read user input from the terminal. When called, it pauses program execution, displays an optional prompt string to guide the user, and waits for the user to type something and press Enter. The function then returns whatever was typed as a string. This means if you ask for a number and want to do math, you must explicitly convert the result using int() or float().

Important note about input(): Before Python 3, there was a function called raw_input() that worked like modern input(), while input() actually evaluated the input as Python code. This security nightmare was fixed in Python 3 — now input() always returns a string safely. Never use eval(input()) — it recreates the old dangerous behavior and can allow malicious code execution.

The print() function outputs text to the console. It is remarkably flexible. You can pass multiple arguments separated by commas, and print() will display them with a space between each: print("The answer is", 42) outputs The answer is 42. The function also accepts two important keyword parameters:

  • sep — the separator between arguments (default is a space): print(1, 2, 3, sep='-') outputs 1-2-3
  • end — what to print at the end (default is a newline \n): print("Loading", end="...") keeps the cursor on the same line

These parameters are incredibly useful for formatting output. For example, to print items without spaces: print("item1", "item2", "item3", sep=""). To print on the same line: print(".", end="") in a loop creates a progress indicator.

f-strings (formatted string literals) are the modern and recommended way to embed expressions inside strings. Prefix the string with f and use curly braces for variables or expressions: print(f"Hello {name.upper()}! You are {age} years old."). You can put any valid Python expression inside the braces: arithmetic, function calls, even conditionals. F-strings are faster, more readable, and less error-prone than the older approaches:

  • Concatenation: "Hello " + name + "!" — clunky with multiple variables
  • % formatting: "Hello %s!" % name — easy to mismatch types
  • .format(): "Hello {}!".format(name) — verbose for complex cases

Multi-line output: Use triple quotes to print strings that span multiple lines. Everything between the triple quotes, including newlines, is preserved exactly as written. This is useful for formatted reports, ASCII art, or any structured text output.

Common mistake: Forgetting to convert input to a number. If you write age = input("Age: ") then age + 1, Python will raise a TypeError because you cannot add a string and an integer. Always convert numeric input explicitly.

# Taking user input
name = input("What is your name? ")
print("Hello, " + name + "!")

# Input returns a string — convert if needed
age = int(input("How old are you? "))
print("Next year you will be", age + 1)

# Formatted output with f-strings
print(f"Nice to meet you, {name}! You are {age} years old.")

# Multi-line output
print("""
  Line 1
  Line 2
  Line 3
""")
Try it: Modify the code to ask for two numbers and print their sum. Then try using sep="\n" to print each item on a separate line. Experiment with different end values.
