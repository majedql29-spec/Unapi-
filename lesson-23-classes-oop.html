Lesson 23: Classes & OOP

Advanced⏰ 12 min

Object-Oriented Programming (OOP) organizes code into classes (blueprints) and objects (instances). Instead of writing functions that operate on separate data structures, you bundle data and behavior together. A Dog class defines what all dogs have in common; my_dog = Dog("Rex") creates a specific dog object. Python supports full OOP with a clean, intuitive syntax.

The __init__ method and self

The __init__ constructor runs automatically when you create an object. The self parameter refers to the current instance and is used to access attributes and methods. It is automatic — Python passes it for you when calling methods: my_dog.bark() becomes Dog.bark(my_dog) internally.

class Dog:
    def __init__(self, name):
        self.name = name
    def bark(self):
        return f"{self.name} says woof!"

my_dog = Dog("Rex")
print(my_dog.bark())

Inheritance and super()

Inheritance lets you create specialized classes from general ones. A subclass inherits all methods and attributes from its parent class and can override or extend them. Use super() to call the parent class's methods:

class Animal:
    def __init__(self, name):
        self.name = name
    def speak(self):
        raise NotImplementedError

class Dog(Animal):
    def speak(self):
        return f"{self.name} says Woof!"

class Cat(Animal):
    def __init__(self, name, toy):
        super().__init__(name)  # Call parent constructor
        self.toy = toy

@classmethod vs @staticmethod

Both are decorators for methods that don't operate on instances. @classmethod receives the class (cls) as the first argument and can access class-level attributes. @staticmethod receives neither self nor cls — it's just a regular function placed inside a class for logical grouping.

The @property decorator

Allows you to define methods that can be accessed like attributes, with optional computed values and validation:

class Circle:
    def __init__(self, radius):
        self._radius = radius
    @property
    def area(self):
        return 3.14 * self._radius ** 2
    @property
    def radius(self):
        return self._radius
    @radius.setter
    def radius(self, value):
        if value < 0:
            raise ValueError("Radius cannot be negative")
        self._radius = value

Dunder methods (special methods)

Double-underscore methods let your objects interoperate with Python's built-in operations. Key ones include: __str__ (called by print()), __repr__ (developer-friendly representation), __len__ (called by len()), __eq__ (==), __lt__ (<), __getitem__ (obj[key]), __add__ (+), __bool__ (truthiness). Implementing these makes your classes feel like built-in types.

Encapsulation and name mangling

Python uses conventions rather than strict access control. A single underscore prefix (_hidden) means "internal use" (convention, not enforced). Double underscore prefix (__private) triggers name mangling to _ClassName__private, making accidental access harder.

Common mistakes

Forgetting self as the first parameter of instance methods — causes TypeError. Using @staticmethod when @classmethod is needed (or vice versa). Not calling super().__init__() in subclasses, leaving parent attributes uninitialized.

Best practices

Use properties instead of getter/setter methods — they're more Pythonic. Favor composition over inheritance ("has-a" is often better than "is-a"). Keep classes focused on a single responsibility. Use __repr__ to provide useful debugging information. Use __slots__ to save memory when creating many instances of a class.

Real-world use case

Modeling a banking system: Account base class with CheckingAccount and SavingsAccount subclasses. The base class handles common logic (balance, owner, account number), while subclasses implement specific rules (withdrawal fees, interest rates). @property provides computed balance, @classmethod creates accounts from CSV data, and dunder methods enable sorting by balance or comparing accounts.

Try it: Add a __str__ method to the Dog class that returns a friendly description. Then create a subclass Puppy(Dog) with a modified bark() that includes a cute suffix. Add a @property that returns the dog's age in human years. Finally, try super() to extend the parent's __init__.
