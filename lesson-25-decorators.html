Lesson 25: Decorators

Advanced⏰ 10 min

Decorators let you modify or extend the behavior of functions or classes without changing their source code. They are widely used in Python web frameworks (Flask, Django), testing tools, and libraries. A decorator is a function that takes another function as an argument, wraps new behavior around it, and returns the modified function. The @decorator syntax is syntactic sugar: @timer def my_func(): pass is equivalent to my_func = timer(my_func).

def timer(func):
    def wrapper():
        print("Starting...")
        result = func()
        print("Done!")
        return result
    return wrapper

@timer
def say_hello():
    print("Hello, JometCode!")

say_hello()

Preserving metadata with @wraps

When you write a decorator, the wrapper function replaces the original — this hides the original function's name, docstring, and signature. Use @functools.wraps(func) on the wrapper to copy this metadata:

from functools import wraps

def logger(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        print(f"Calling {func.__name__}")
        return func(*args, **kwargs)
    return wrapper

Always use @wraps in your decorators — it preserves the original function's __name__, __doc__, __module__, and __annotations__.

Decorators with arguments

If you need to parameterize a decorator, add an outer layer:

def repeat(n):
    def decorator(func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            for _ in range(n):
                func(*args, **kwargs)
            return wrapper
        return decorator

@repeat(n=3)
def say_hi():
    print("Hello!")

The pattern is decorator_factory(arguments) -> decorator -> wrapper. This triple-nesting is essential for working with Flask route decorators like @app.route("/").

Class-based decorators

You can also implement decorators as classes using __call__:

class CountCalls:
    def __init__(self, func):
        @wraps(func)
        self.func = func
        self.count = 0
    def __call__(self, *args, **kwargs):
        self.count += 1
        print(f"Call #{self.count} of {self.func.__name__}")
        return self.func(*args, **kwargs)

Real-world examples

Timing — measure and log execution time of functions for performance profiling. Logging — automatically log every function call with arguments and return values. Caching (memoization) — store results of expensive computations (@functools.lru_cache). Authentication — check user permissions before executing a view function in web frameworks. Rate limiting — restrict how often a function can be called within a time window.

Common mistakes

Forgetting to use @wraps — breaks introspection and documentation tools. Forgetting to return the wrapper function from the decorator. Not forwarding *args and **kwargs from the wrapper to the original function. Stacking decorators in the wrong order — remember they apply bottom-up (nearest to the function first).

Best practices

Always use @wraps to preserve function metadata. Forward *args, **kwargs in all wrappers. Keep decorators focused on one concern. Document what the decorator does and any side effects. Use functools.lru_cache instead of writing your own memoization.

Try it: Add @wraps to the existing timer decorator, then print say_hello.__name__ to confirm the original name is preserved. Then create a new decorator that logs the arguments and return value of the decorated function. Finally, stack two decorators on one function and observe the execution order.
